RESP介绍

简介：

RESP(REdis Serialization Protocol) 是Redis提供的文本协议，协议中 所有发送至 Redis 服务器的参数都是二进制安全（binary safe）,Redis Client 通过TCP来建立与 Redis Server 的连接，默认端口号6379。

协议格式

例子：set key value
对应的协议：

对应的文本：*3\r\n$3\r\nset\r\n$3\r\nkey\r\n$5\r\nvalue\r\n

响应信息

Redis 命令会返回多种不同类型的回复。
通过检查服务器发回数据的第一个字节， 可以确定这个回复是什么类型：

  • 状态回复（status reply）的第一个字节是 “+”

  • 错误回复（error reply）的第一个字节是 “-“

  • 整数回复（integer reply）的第一个字节是 “:”

  • 批量回复（bulk reply）的第一个字节是 “$”

  • 多条批量回复（multi bulk reply）的第一个字节是 “*”
