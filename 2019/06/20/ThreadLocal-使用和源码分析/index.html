线程本地存储

线程本地存储为使用相同变量的每个不同的线程创建不同的存储，例如有 5 个线程都要使用变量 local所表示的对象，那么本地线程存储就会生成 5 个用于 local 的不同的存储块，它可以使状态和线程关联起来, 总而言之就是: 保存线程上下文信息，在该线程任意需要的地方可以获取.

在 Thread 中定义：

也就是说每个线程有一个自己的ThreadLocalMap ，所以每个线程往这个ThreadLocal中读写隔离的，并且是互相不会影响的
ThreadLocalMap 是 ThreadLocal的静态内部类，

弱引用 也是用来描述非必需对象的，当JVM进行垃圾回收时，无论内存是否充足，该对象仅仅被弱引用关联，那么就会被回收。

ThreadLocal 主要方法：
从 Entry 的源码可以看到Entry的key是l弱引用，ThreadLocal被垃圾回收后，在ThreadLocalMap里对应的Entry的键值会变成null，但是Entry.value是强引用，那么Entry里面存储的Object，并没有办法进行回收，所以需要手动调用ThreadLocal.remove()方法，手动释放对象

set 设置当前线程的需要保存的值

get 获取当前线程已经保存的值，如果没有设置则从initialValue方法中获取初始值

最佳实践

  • 应该在我们不使用的时候，主动调用remove方法进行清理。

  • 把ThreadLocal定义为static，由于ThreadLocal有强引用在，那么在ThreadLocalMap里对应的Entry的键会永远存在，那么执行remove的时候就可以正确进行定位到并且删除！

在阿里 Java 规范中描述:

ThreadLocal 对象使用 static 修饰，ThreadLocal 无法解决共享对象的更新问题。 说明:这个变量是针对一个线程内所有操作共享的，所以设置为静态变量，所有此类实例共享此静态变 量，也就是说在类第一次被使用时装载，只分配一块存储空间，所有此类的对象(只要是这个线程内定义 的)都可以操控这个变量。
